Why test your site with PageGauge?

Read FAQ

  • How does PageGauge work?

    We compiled the main criteria for website usability from multiple sources and identified the most important. From these we assessed how easy there were to programmatically assess and selected the the most impactful. We then built algorithms to test these things (e.g. is the site responsive, or how many layers deep is it) and assigned scoring criteria for them. When you submit a site to PageGauge we run these tests and present you with your score.

  • How accurate is it?

    It’s extremely accurate to the parameters that we have set - which is why it’s more reliable than subjective analysis by humans. BUT the analysis isn’t very sophisticated, and definitely not as good a proxy for usability as real user tests. We hope to improve on the sophistication of analysis in future, but this is a side project that we wanted to launch and iterate over time.

  • Why is it free?

    This is a side-project by the Chameleon team, designed and built in a single day. Our mission is improved interface usability and while our main product is for improving user onboarding for web applications, we felt this tool could be valuable for lots of teams. We are not charging for this because we don’t have the capacity to handle the responsibility that would bring.

  • How can I make a suggestion?

    We would love to hear feedback and we may make some improvements in future. If you have ideas or would even like to participate a v2.0 hackathon please email us at hello@pagegauge.io.
